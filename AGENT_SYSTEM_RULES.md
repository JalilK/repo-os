# Repo OS System Rules

Every repo generated by this system must teach the LLM to prefer ACP commands over raw shell for recurring repo operations.

## Mandatory rule

If a requested task fits an existing ACP command, use the ACP command.

If a requested task is workflow-critical and repeated, add or extend an ACP command.

Do not let repeated repo operations remain ad hoc terminal snippets.

## Goal

Move repeated work upward into stable commands so future sessions are faster, safer, and more deterministic.

## Required generated layers

- ACP artifact skeleton
- repo policy layer
- verification layer
- ACP CLI wrapper
- command-first execution rules
- source-of-truth indexing

## Required generated repo behavior

Generated repos must support these local commands

- `acp status`
- `acp start <feature-name>`
- `acp verify`
- `acp pr body`
- `acp pr create`
- `acp complete`
- `acp command suggest "<task description>"`
